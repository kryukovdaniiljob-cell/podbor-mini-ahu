# Задача для Claude Code: веб-сервис подбора КПВУ (компактных приточно-вытяжных установок)

## Что нужно сделать
Построить **полностью рабочий веб-сервис подбора КПВУ**, точно воспроизводящий расчётный движок Excel-калькулятора `MiniAHU_Selector_v_2_3_2025.xlsx`. Пользователь вводит параметры, сервис подбирает установку, считает все технические характеристики и строит **график аэродинамических характеристик** (кривая вентилятора + кривая сети + рабочая точка). Все результаты должны **совпадать с Excel** (см. контрольный пример в конце).

Вместе с этим промтом тебе передаётся файл **`models_data.json`** — это все данные из Excel (24 модели, 157 типоразмеров, коэффициенты вентиляторных кривых, габариты, КПД рекуператоров, психрометрические коэффициенты, узлы MST). **Используй его как единственный источник данных — не придумывай числа.**

## Технологический стек (рекомендация)
- **Frontend:** React + Vite + TypeScript, Tailwind для стилей.
- **График:** Recharts (ScatterChart + линии) или Chart.js. Должна быть возможность экспорта в PDF/печать спецификации.
- **Логика расчёта:** вся в чистом TypeScript-модуле `engine.ts` (без бэкенда — расчёт целиком на клиенте). Данные грузятся из `models_data.json`.
- Опционально лёгкий бэкенд не нужен: всё работает офлайн в браузере.
- Язык интерфейса — **русский**.

## 2. Входные параметры (форма)

Сгруппировать на форме. В скобках — соответствие ячейкам Excel и допустимые значения/диапазоны.

**Тип и режим подбора**
- `installation_type` — Тип установки (R5): `приточная` | `приточно-вытяжная`.
- `selection_mode` — Способ подбора (R6): `автоматический` | `вручную`.
- Если `вручную`:
  - `manual_model_supply` (S6) — для приточной: один из `CAU_F, CAU_W, ECO_A, ECO_Slim, ECO_Slim_W, Airtube, Swift`.
  - `manual_model_se` (T6) — для приточно-вытяжной: один из `Nova, Unimax_*`.
  - `manual_size_no` (U6) — № типоразмера, целое 1…35.

**Воздушные параметры приток**
- `flow` — Расход, м³/ч (R7), целое ≥ 1.
- `head` — Напор (внешнее сопротивление сети), Па (R8), 1…1100.
- `t_outdoor` — t наружного воздуха, °C (R10), −60…+30.
- `rh_outdoor` — φ наружного воздуха, % (S10), 1…100.
- `t_supply` — требуемая t приточного воздуха, °C (R11), 0…+50.

**Воздушные параметры вытяжка** (только если `приточно-вытяжная`)
- `t_indoor` — t внутреннего/вытяжного воздуха, °C (R12), 0…+50.
- `rh_indoor` — φ внутреннего воздуха, % (S12), 1…100.

**Нагреватель**
- `heater_type` — Тип нагревателя (R9): `водяной` | `электрический` | `без нагревателя`.
- Если `водяной`:
  - `t_water_in` — t воды на входе, °C (R18), 40…90.
  - `t_water_out` — t воды на выходе, °C (R19).

**Рекуператор** (приточно-вытяжная)
- `recup_type` — Тип рекуператора (R13): `роторный` | `пластинчатый`.

**Конструктив / опции** (влияют только на автоподбор модели)
- `case_type` — Тип корпуса (R14): `изолированный` | `не изолированный`.
- `automation` — Автоматика (R15): `встроенная` | `внешняя`.
- `motor_type` — Тип двигателя (R16/R22): `ЕС` | `асинхронный`.
- `wall_thickness` — Толщина (R17): `уменьшенная` | `стандартная`.
- `air_outlet` — Расположение выхода (R20): `вверх` | `вбок`.
- `mounting` — Монтаж (R21): `подвесная` | `напольная`.

## 3. Автоматический выбор МОДЕЛИ (имя листа `B26`)

Реализовать ровно как в Excel (`Расчеты!Q30` для приточной, `Расчеты!R30` для приточно-вытяжной).

**Приточная (`B26 = Q30`):**
```
if mode == "вручную": model = manual_model_supply
elif heater_type == "водяной":
    model = (automation == "встроенная") ? "ECO_Slim_W" : "CAU_W"
elif heater_type == "без нагревателя":
    model = "CAU_F"
elif case_type == "изолированный":
    if automation == "встроенная":
        if motor_type == "ЕС": model = "Swift"
        else: model = (wall_thickness == "стандартная") ? "ECO_A" : "ECO_Slim"
    else: model = "CAU_F"
else:  # не изолированный
    model = "Airtube"
```

**Приточно-вытяжная (`B26 = R30`):**
```
if mode == "вручную": model = manual_model_se
elif heater_type == "без нагревателя": model = "Nova"
else:
    p = "Unimax_"
    p += (recup_type == "пластинчатый") ? "P_" : "R_"
    if recup_type == "пластинчатый":
        if air_outlet == "вбок": p += (mounting == "напольная") ? "S" : "C"
        else: p += "V"
    else:  # роторный
        p += (air_outlet == "вбок") ? "S" : "V"
    p += (heater_type == "водяной") ? "W" : "E"
    if recup_type == "пластинчатый" and motor_type == "ЕС": p += "_EC"
    model = p
```

## 4. Выбор ТИПОРАЗМЕРА (две позиции: `M60` и `M61`)

Для выбранной модели перебрать все типоразмеры. Обозначим запрос `Qreq=flow`, `Hreq=head`.

**4.1 Фильтр пригодности (Excel колонка AM):**
Типоразмер *проходит*, если кривая вентилятора в точке запроса даёт напор не меньше требуемого:
```
P_fan(Qreq) = a*Qreq^2 + b*Qreq + c + d*Qreq^3
qualifies = (P_fan(Qreq) - Hreq) >= 0
```
Непрошедшим присваиваем «штраф» (∞).

**4.2 `M60` — типоразмер по аэродинамике (Excel AN1):**
```
если mode=="вручную": M60 = manual_size_no   (индекс в массиве sizes = manual_size_no - 1)
иначе если ни один не прошёл: ОШИБКА "Слишком большой расход/напор!"
иначе: M60 = тот, у кого МИНИМАЛЬНЫЙ score_O среди прошедших
        (при равных score_O берём первый по порядку — это база с наименьшим нагревателем)
```
`M60` задаёт **коэффициенты вентиляторной кривой** (fan_a..fan_d), **КПД рекуператора** (recup_eff_T/H) и преднагрев — то есть всё для графика и теплорасчёта рекуператора.

**4.3 `M61` — типоразмер полной комплектации (Excel AN2):**
Среди типоразмеров, имеющих **тот же** `score_O`, что и у `M60` (одно базовое «семейство» по производительности), выбрать вариант с **наиболее подходящим нагревателем** — минимальным неотрицательным избытком мощности:
```
required_heater_kW = R28  (см. §6.1)
surplus(i) = heater_power_kW(i) - required_heater_kW
M61 = argmin по surplus, где surplus >= 0 (округление избытка вверх до целого, как ROUNDUP);
      если ни один не покрывает — берём с минимальным |surplus|*1000 (т.е. ближайший);
для heater_type=="без нагревателя": M61 = M60.
для mode=="вручную": M61 = manual_size_no.
```
`M61` задаёт фильтры, ток, мощность, габариты, наименование, напряжение и т.д. в спецификации.

> Замечание: у приточных с электронагревом (CAU_F, ECO_A…) одна базовая установка имеет несколько вариантов по мощности ТЭНов — `M61` выбирает нужный ТЭН. У приточно-вытяжных нагреватель часто отдельный, тогда `M60==M61`.

## 5. Аэродинамический расчёт и ГРАФИК (ключевая часть)

Используем коэффициенты `M60`. Обозначим `k = Hreq / Qreq^2` (наклон параболы сети).

**5.1 Кривая вентилятора (Хар-ка установки):**
```
P_fan(Q) = fan_a*Q^2 + fan_b*Q + fan_c + fan_d*Q^3
```
Построить по сетке расходов Excel: `Q ∈ {0,10,20,30,40,50,60,80,100,150,200,250,300,...,1000, шаг 100 до 2000, далее шаг 200 до 7000}`.
(Полная сетка — в Excel `Расчеты!M65:M126`; можно просто взять плотную сетку 0…Qmax с шагом 10–20 для гладкости, результат идентичен.)

**5.2 Кривая сети (Хар-ка сети):** `P_sys(Q) = k*Q^2` на той же сетке.

**5.3 Рабочая точка (пересечение):**
Найти `Q_op` — корень `P_fan(Q) − P_sys(Q) = 0` (наибольший положительный корень в рабочей зоне). В Excel это делается грубым перебором + уточнением (M65:S126); в TS достаточно:
- пройти по плотной сетке, найти смену знака `P_fan−P_sys`, уточнить бинарным поиском/линейной интерполяцией. Должно дать `Q_op` с точностью до ~0.1 м³/ч.

**5.4 Фактический расход/напор (рабочая точка на графике):**
```
actual_flow (E25) = min(Qreq, Q_op)
   // нюанс Excel: для моделей CAU* и Airtube при «рекуператор = нет» E25 = Q_op (без min);
   //  в обычном случае и для всех остальных E25 = min(Qreq, Q_op). По умолчанию используем min.
actual_head (E26) = actual_flow^2 * k = actual_flow^2 * Hreq / Qreq^2
```

**5.5 Серии графика (scatter, оси: X=Расход, м³/ч; Y=Напор, Па):**
1. **«Хар-ка установки»** — линия `(Q, P_fan(Q))` по сетке. (Excel: УстХ/УстУ.)
2. **«Хар-ка сети»** — линия `(Q, P_sys(Q))` по сетке. (СетьХ/СетьУ.)
3. **«Рабочая точка»** — маркер `(actual_flow, actual_head)`. (E25,E26.)
4. **«Данные запроса»** — маркер `(Qreq, Hreq)`. (C9,E9.)
5. Вспомогательные пунктирные линии-выноски от рабочей точки к осям (вертикаль `x=actual_flow` от 0 до actual_head; горизонталь `y=actual_head` от 0 до actual_flow) — как в Excel (серии из Расчеты!R59:S61).

Сделать график крупным, с подписями осей, легендой, сеткой. Кривая установки — сплошная; сеть — другого цвета; точки — заметные маркеры с подписью координат.

## 6. Теплотехнический расчёт (психрометрия)

**6.0 Давление насыщенного пара `Psat(T)`, кПа** (`Свойства_воздуха`):
```
if T < 0:  Psat = e*T^4 + d*T^3 + a*T^2 + b*T + c   // коэф. sat_pressure_below0_kPa
else:      Psat =        d*T^3 + a*T^2 + b*T + c     // sat_pressure_above0_kPa
```
**Влагосодержание** `d(T, φ)` г/кг, φ в %: `d = 622 / (101.3 / (φ/100 * Psat(T)) - 1)`
**Энтальпия** `h(T, d)` кДж/кг, d в г/кг: `h = 1.01*T + (2500 + 1.8*T) * d/1000`

**6.1 Требуемая мощность нагревателя `R28`, кВт:**
- Приточная: `Q_heat = E25/3600 * 353.5/(273 + t_supply) * (t_supply - t_outdoor) * 1.01`
- Приточно-вытяжная: то же, но вместо `t_outdoor` берётся температура воздуха **после рекуператора** `J33` (см. 6.3).
- `R28 = max(Q_heat, 0)`.

**6.2 Номинальная мощность `R29`:** электрический → `heater_power_kW(M61)`; водяной → мощность водяного узла (см. §7); без нагревателя → 0.

**6.3 Рекуператор (приточно-вытяжная), температуры на выходе:**
Температура приточного воздуха после рекуператора (с учётом преднагрева `CM` при M60):
```
t_after_recup_supply (I33) = min( t_outdoor + CM/1.005/flow/353.5*(273+t_supply)*3600 , t_supply )
t_after_recup_with_eff (J33) = min( I33 + recup_eff_T(M60) * (t_indoor - I33_базовая) , t_supply )
```
(точную формулу J33 см. в Excel `Лист_подбора!J33`; реализовать 1:1). Эффективность по температуре `J36 = (J33 − I33)/(t_indoor − I33) * 100 %`. Энтальпийная эффективность — по `recup_eff_H` и энтальпиям (формула в Расчеты!T25/Лист_подбора!L36).
Температуры/влажности вытяжного тракта (вход/выход рекуператора) — по блоку `Расчеты!Q24:T26` и `Лист_подбора!I31:L36`. Реализовать таблицу «Рекуператор» как в спецификации Excel.

**6.4 Достижимая t на выходе нагревателя `R31`:**
- электрический: `R31 = heater_power_kW/353.5*(t_supply+273)/E25*3600 + (приточная? t_outdoor : J33)`
- водяной: из теплового расчёта водяного калорифера.

## 7. Водяной нагреватель и смесительный узел MST (если `heater_type=="водяной"`)

- Расход воды `L29` = `min(номин, треб) / 4.2 * 3.6 / (t_water_in − t_water_out)` (м³/ч).
- Потери давления по воде `L30` = коэф.×L29². 
- Подбор узла MST: для каждого узла из `mst_units` считать `ΔP_клапан=(L29/Kvs)^2*100`, авторитет `=ΔP/(ΔP+L30)`, статический напор узла `H=a*L29^2+b*L29+c`; выбрать узел с максимальным критерием `J` (как `Подбор_MST!J2:J10`, формула `IF((H−L30)*авторитет<0,0,(H−L30)*авторитет)`, авторитет учитывается только если ≥ 1/1.428). Вернуть марку узла.
- Для приточной с водяным нагревом модель = `CAU_W`/`ECO_Slim_W`; потери по воде/воздуху калорифера берутся из колонок CAU_W (AW/AX) через VLOOKUP по наименованию (см. `Лист_подбора!K30/L30`).

> Узел MST подбирается **только** при водяном нагреве; иначе блок скрыт/прочерк.

## 8. Итоговая спецификация (правый/нижний блок результата)

Воспроизвести таблицу `Лист_подбора` (строки 1–41). Минимум вывести:
- **Полное наименование** (`A7`): `name(M61)`; если ни один типоразмер не подобран — «Не удалось подобрать установку!».
- **Заголовок** (`E3`): «Компактная <приточная|приточно-вытяжная> установка».
- **Данные запроса:** расход, напор, t нар., t прит., (для П-В: t внут., φ внут.).
- **Параметры установки:** расход (E25), напор (E26), тип фильтра приток/вытяжка, тип нагревателя, потребляемая мощность, рабочий ток, напряжение, частота вращения вентилятора(ов).
- **Нагреватель:** требуемая (R28) и номинальная (R29) мощность, t на входе/выходе по воздуху и (для водяного) по воде, расход воды, потери давления.
- **Рекуператор** (П-В): температуры/влажности вход-выход приток и вытяжка, мощность, эффективность по температуре и по энтальпии.
- **Массогабаритные характеристики, мм:** W,W1,W2,W3,H,H1,H2,H3,H4,L,L1,L2,L3,L4,D,d и Вес, кг (из `dims` для M61).
- **Комплектация / автоматика** (`accessories`): тип регулятора, клапан, соединители, привод, шумоглушитель, датчик давления и т.п.

## 9. Предупреждения / валидация (блок `O34:O40`)

Показывать сообщения (как в Excel), блокировать «успешный подбор» при критичных:
- Нет подходящего типоразмера → **«Слишком большой расход/напор!»** (никакой типоразмер не прошёл фильтр §4.1).
- `R28 > R29` → **«Недостаточная мощность нагревателя!»**
- «без нагревателя» недоступна для крупных типоразмеров (CAU_F: M61>11; иначе >3) → соответствующее сообщение.
- `t_supply − t_outdoor < 1` → «Слишком низкая температура приточного воздуха!».
- Для П-В: `t_indoor − t_outdoor < 10` → «Слишком маленькая температура вытяжного воздуха!».
- При очень низкой t (приточная, водяной, t_outdoor<−30) → «Необходимо предусмотреть преднагрев!»; П-В при t после рекуператора <−30 → «Необходимо предусмотреть внешний преднагрев!».
- Ручной режим: № типоразмера больше максимального для модели → «Слишком большой номер типоразмера…».
- Соблюдать диапазоны полей ввода из §2 (валидация формы с подсказками).

## 11. КОНТРОЛЬНЫЙ ПРИМЕР (обязательно совпасть)

Ожидаемые результаты движка:
- Коэффициенты вентилятора M60: `a=0.0002537, b=-0.6308, c=694.5, d=-8.66e-08`.
- `P_fan(0) = 694.5 Па`; `P_fan(500) ≈ 431.7 Па ≥ 150` → проходит.

- **Фактический расход `actual_flow = min(500,742) = 500 м³/ч`** (Excel `E25 = 500`).
- **Фактический напор `actual_head = 500²·0.0006 = 150 Па`** (Excel `E26`).
- Психрометрия наружного воздуха (t=−30, φ=80%): энтальпия `≈ −29.84 кДж/кг`, влагосодержание `≈ 0.19 г/кг` (Excel `Расчеты!Q25/R32`).

Написать автотест, проверяющий эти значения с допуском ±1%.
