# Сервис подбора КПВУ

Веб-сервис подбора компактных приточных и приточно-вытяжных установок,
воспроизводящий расчётный движок Excel-калькулятора `MiniAHU_Selector_v_2_3`.

## Запуск

```bash
npm install
npm run dev      # режим разработки  → http://localhost:5173
npm run build    # production-сборка → dist/
npm test         # автотесты движка (контрольный пример §11)
```

## Архитектура

```
src/
  data/models_data.json   единственный источник данных (24 модели, 157 типоразмеров)
  engine/
    psychro.ts            Psat, влагосодержание, энтальпия (§6.0)
    fan.ts                кривая вентилятора/сети, рабочая точка, сетка графика (§5)
    selectModel.ts        автоподбор модели B26 = Q30/R30 (§3)
    selectSize.ts         выбор типоразмеров M60 и M61 (§4)
    recuperator.ts        температуры и КПД рекуператора (§6.3)
    heater.ts             электр./водяной нагрев, узел MST (§6.1–6.4, §7)
    engine.ts             оркестратор: вход → полный результат
    types.ts
  components/
    InputForm.tsx         форма ввода с валидацией (§2)
    AeroChart.tsx         график аэродинамики (Recharts) (§5)
    SpecSheet.tsx         итоговая спецификация (§8), печать/PDF
    Warnings.tsx          предупреждения (§9)
  App.tsx                 реактивный пересчёт при любом изменении формы
tests/engine.test.ts      контрольный пример + автоподбор + психрометрия
```

## Контрольный пример (§11)

Вход по умолчанию (Unimax_P_CE, типоразмер №4) даёт результаты, совпадающие с Excel
(допуск ±1 %): Q_op ≈ 742 м³/ч, фактический расход 500 м³/ч, напор 150 Па,
требуемая мощность нагревателя 4.23 кВт, t после рекуператора −4.08 °C,
энтальпия наружного воздуха −29.84 кДж/кг. Все эти значения проверяются автотестами.

Все числа берутся только из `models_data.json` — «магических» констант в коде нет.
